## Что сделано сейчас
Текущая версия — статический сайт (`index.html`, `services.html`, `styles.css`, `animations.js`).

## Цель переноса на фреймворки
Перенести сайт на фреймворки **без поломки текущей версии**, с выделенной админкой:
- редактирование контактной информации;
- редактирование каталога услуг;
- просмотр заявок пользователей;
- вход в админку по прямой ссылке + логин/пароль.

---

## Выбранный стек (простой для новичка и подходящий под задачу)

### База данных
- **PostgreSQL (SQL)**

Почему: надежная SQL-база, хорошо подходит для каталога услуг, контактов и заявок, простые связи между сущностями.

### Бэкенд
- **FastAPI + SQLAlchemy (Python)**

Почему: понятный синтаксис, быстрая разработка CRUD, удобная документация API из коробки.

### Фронтенд
- **Vue (Vite)**

Почему: мягкий порог входа для новичка после обычного HTML/CSS/JS, удобен для постепенного переноса страниц в компоненты.

## Кратко по сущностям и админке
Подробности в `deepwiki/entities-and-admin.md`.

Сущности:
1. `AdminUser` — администратор (логин/пароль).
2. `ContactInfo` — контактные данные сайта.
3. `ServiceCategory` — категория услуг.
4. `ServiceItem` — услуга в каталоге.
5. `UserRequest` — заявка от пользователя.

Что редактируется в админке:
- `ContactInfo` — полностью редактируется;
- `ServiceCategory`/`ServiceItem` — полный CRUD;
- `UserRequest` — просмотр, смена статуса, комментарий менеджера (без удаления для истории).

## План аккуратного переноса (без поломки)
1. Поднять новый backend и frontend отдельно от legacy.
2. Повторить текущую верстку в Vue-компонентах (визуально 1-в-1).
3. Подключить API для каталога/контактов/формы заявок.
4. Добавить `/admin` со входом по логину и паролю.
5. Протестировать и только потом переключить основной деплой на новую версию.

Подробный пошаговый план: `deepwiki/migration-plan.md`.
