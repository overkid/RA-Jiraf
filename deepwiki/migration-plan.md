# План переноса сайта на фреймворки без поломки

## Этап 1. Подготовка
1. Оставить текущую статическую версию как `legacy`.
2. Поднять PostgreSQL.
3. Создать каркас FastAPI + SQLAlchemy + Alembic.
4. Создать каркас Vue (Vite + Vue Router + Pinia).

## Этап 2. Бэкенд
1. Реализовать модели: `AdminUser`, `ContactInfo`, `ServiceCategory`, `ServiceItem`, `UserRequest`.
2. Добавить миграции БД.
3. Реализовать API:
   - публичные: контакты, каталог услуг, отправка заявки;
   - админские: CRUD контактов/каталога, просмотр и обновление статусов заявок.
4. Реализовать авторизацию `/admin/login` (логин + пароль).

## Этап 3. Фронтенд
1. Перенести текущие страницы в Vue-компоненты 1-в-1 по внешнему виду.
2. Подключить публичный API для данных.
3. Сделать страницу `/admin` и вложенные страницы:
   - `/admin/contacts`
   - `/admin/services`
   - `/admin/requests`
4. Реализовать guard для закрытых маршрутов.

## Этап 4. Проверка
1. Сверить внешний вид новой версии с legacy.
2. Проверить отправку заявок.
3. Проверить логин в админку.
4. Проверить редактирование контактов и услуг.
5. Проверить просмотр/статусы заявок.

## Этап 5. Переключение
1. Деплой новой версии в staging.
2. Финальный smoke-test.
3. Переключение production на новую версию.
4. Legacy оставить как резерв на короткий период.
